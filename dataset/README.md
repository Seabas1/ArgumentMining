# RuArgMining — разметка аргументов в русскоязычных правовых текстах

Датасет предложений из российских правовых документов, размеченных по ролям в аргументации.

## Описание

Каждое предложение относится к одному из пяти классов аргументации. Датасет создан автоматической разметкой через LLM с последующей ручной проверкой спорных случаев.

**Домен:** судебные решения, правовые акты, письма Минфина и ФНС по налоговым вопросам  
**Язык:** русский  
**Объём:** 11 865 предложений, 427 документов

## Метки

| Метка | Описание | Примеры маркеров | Примеров | Доля |
|-------|----------|-----------------|---------|------|
| `CLAIM` | Позиция, требование или вывод стороны / суда / органа | «полагает», «суд приходит к выводу», «обязать» | 5900 | 49.7% |
| `EVIDENCE` | Ссылка на норму, документ или факт как доказательство | «согласно ст.», «на основании», «что подтверждается» | 3827 | 32.3% |
| `NON_ARG` | Процедурный текст без аргументации | даты заседаний, явка сторон, реквизиты | 1773 | 14.9% |
| `PREMISE` | Логическое обоснование без внешней ссылки | «поскольку», «так как», «следовательно» | 222 | 1.9% |
| `REBUTTAL` | Прямое опровержение чужой позиции | «однако», «суд не соглашается», «отклоняется» | 143 | 1.2% |

## Формат

Файл `annotations.jsonl` — каждая строка один JSON-объект:

```json
{
  "doc_id": "pismo-minfina-2022-03-17",
  "position": 4,
  "text": "Поскольку ответчик допустил просрочку, истец вправе требовать неустойки.",
  "label": "PREMISE",
  "confidence": 0.93,
  "reasoning": "Логическая цепочка «поскольку X → вправе Y» без ссылки на внешний источник.",
  "disputed": false,
  "model": "qwen3_5-9b_latest",
  "source_file": "pismo-minfina-2022-03-17.jsonl"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `doc_id` | string | Идентификатор документа |
| `position` | int | Порядковый номер предложения в документе |
| `text` | string | Текст предложения |
| `label` | string | Метка аргументации |
| `confidence` | float | Уверенность модели (0.0–1.0) |
| `reasoning` | string | Объяснение выбора метки |
| `disputed` | bool | `true` если confidence < 0.65 — приоритет для ручной проверки |
| `model` | string | Модель, которая размечала |
| `source_file` | string | Исходный файл аннотации |

## Использование

```python
import json
from pathlib import Path

rows = [json.loads(l) for l in Path("annotations.jsonl").read_text(encoding="utf-8").splitlines() if l.strip()]

# Только уверенные метки
confident = [r for r in rows if not r["disputed"]]

# Распределение меток
from collections import Counter
print(Counter(r["label"] for r in rows))
```
